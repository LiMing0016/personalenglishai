# Claude Code 工作流规范

## 编码前确认

每次开始写代码之前，必须先执行以下步骤：

1. **确认需求**：向我简要复述你理解的任务需求，确认无误后再开始
2. **询问是否开始**：明确问我"是否可以开始编码？"，等待我确认后再动手

## 分支管理

开始编码前，必须询问以下问题：

1. 是否需要为本次任务创建新的 Git 分支？
2. 如果需要，建议的分支名称是什么？（格式：`feature/功能名称` 或 `fix/修复内容`）
3. 等待我确认分支策略后再开始编码

## 编码完成后

代码编写完成后，必须依次询问以下问题：

1. **代码合并**：是否需要将当前分支合并到主分支（main/master）？
2. **文档更新**：是否需要将本次新增的功能、修改内容、建议事项写入相应文档？
   - README.md（功能说明）
   - CHANGELOG.md（变更记录）
   - 其他相关文档

## 通用规则

- 不要在未经我确认的情况下自行开始编码
- 不要在未经我确认的情况下自行合并分支
- 不要在未经我确认的情况下自行修改文档
- 每一步操作都需要得到我的明确许可
