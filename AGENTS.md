# 协作规则

- 每次开始新需求前，先评估是否需要新建分支。
- 每次需求完成后，检查是否需要更新项目文档。
- 每次需求完成后，评估当前改动是否可以合并到 `main`。

## 作用域说明

- 根目录 `AGENTS.md` 只负责仓库级协作规则。
- 前端实现规则放在 `web/AGENTS.md`。
- 后端实现规则放在 `backend/AGENTS.md`。
- 修改某一端代码时，优先同时遵守对应目录下的 `AGENTS.md`。

## 通用协作原则

1. 优先做小范围、安全、可验证的修改。
2. 除非明确要求，不要顺手重构无关模块。
3. 修改前先确认影响范围、依赖链路和验证方式。
4. 修改后明确说明实际运行了哪些验证，哪些没有运行。
5. 不要覆盖或回退不属于当前任务的现有改动。

## 文档与合并

- 涉及架构、接口、状态流、缓存策略、部署方式变化时，检查是否需要同步更新项目文档。
- 文档改动本身也要检查是否适合直接合并到 `main`。
- 如果改动是高风险功能开发而不是文档或小修，优先评估是否应在独立分支完成。
