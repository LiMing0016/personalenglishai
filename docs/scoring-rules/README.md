# 评分细则设计目录

本目录专门用于沉淀评分系统的可执行细则，不放战略描述，不放零散讨论。

目标是把“产品上觉得合理”的评分想法，收敛成后端、Prompt、前端解释都能落地的规则设计。

## 目录约定

- `postgrad-exam-policy.md`
  - 第一阶段考研考试评分细则草案
- 后续建议按学段或场景继续拆分
  - `highschool-exam-policy.md`
  - `middle-school-exam-policy.md`
  - `free-mode-weight-policy.md`

## 设计原则

所有评分细则文档都应尽量回答清楚这几件事：

1. 评分对象是谁
2. 满分和档位怎么定义
3. 先判什么，后判什么
4. 哪些因素决定最高档位
5. 哪些因素只影响档内分数
6. 哪些情况必须封顶或扣分
7. 后端需要哪些结构化字段
8. 前端最终向用户解释什么

## 和现有文档的边界

- `docs/scoring-methodology.md`
  - 说明当前系统“已经怎么做”
- `docs/evaluation-system-strategy.md`
  - 说明评分系统“为什么重要、未来往哪走”
- `docs/scoring-rules/*`
  - 说明“下一步准备按什么规则做”

## 当前阶段重点

当前优先沉淀：

- `postgrad` 学段
- `exam` 模式
- “方向门槛 + 质量细化 + 硬约束”评分协议

暂不在一期强判定中纳入：

- 模板痕迹
- 抄袭/雷同
