# 硕士阶段写作 Rubric
> key: `postgrad-exam-v1`
> stage: `postgrad`
> mode: `exam`

## 一、适用范围

本 Rubric 适用于考研英语写作评分首版，覆盖：

- `task1`：应用文 / 功能写作
- `task2`：图画、图表、表格、现象评论类大作文

设计原则：

- 不再沿用高中写作标准
- 保持现有 6 维结构，兼容现有接口与前端
- 对外统一为一套 `postgrad-exam-v1`
- 对内按 `taskType` 注入子规则
- 首版统一使用 `100` 分制输出
- 英语一 / 英语二原始分映射后续再做

---

## 二、分档与系统映射

### 2.1 等级锚点与 `level_score`

| 等级 | level_score | 分数语义 |
|------|-------------|----------|
| A | 90 | 明显高于平均考生，任务完成充分，语言成熟 |
| B | 75 | 达到较好水平，任务基本完成，语言较稳 |
| C | 60 | 达到基本合格线，有明显不足但可读可判 |
| D | 42 | 任务完成差或语言问题较重，质量偏低 |
| E | 20 | 严重跑题、严重未完成任务或语言难以理解 |

说明：

- 这里的 `level_score` 是系统内部计算锚点，不是用户看到的最终总分区间。
- 维度先按 A-E 映射到 `level_score`，再汇总为总分。

### 2.2 总分档位区间

| 档位 | 中文 | 区间 |
|------|------|------|
| Band 5 | 优秀 | 85-100 |
| Band 4 | 良好 | 70-84 |
| Band 3 | 中等 | 55-69 |
| Band 2 | 较差 | 40-54 |
| Band 1 | 差 | 0-39 |

## 三、维度定义

| 维度 | 含义 |
|------|------|
| `content_quality` | 内容展开质量、论述深度、信息有效性 |
| `task_achievement` | 是否完成题目要求的写作任务 |
| `structure` | 篇章组织、段落安排、信息顺序和衔接 |
| `vocabulary` | 词汇准确性、书面性、搭配质量、丰富度 |
| `grammar` | 语法准确性、复杂结构控制能力 |
| `expression` | 表达自然度、正式度、整体文风成熟度 |

---

## 四、维度权重建议

> 权重用于总分汇总，不替代方向门槛、任务封顶、字数处罚等规则。
> 即：先判任务是否完成，再做加权汇总；不能用语言分“补回”严重偏题或未完成任务。

### 4.1 `task2` 大作文

| 维度 | 权重 |
|------|------|
| `task_achievement` | 30% |
| `content_quality` | 25% |
| `structure` | 15% |
| `vocabulary` | 10% |
| `grammar` | 10% |
| `expression` | 10% |

汇总导向：

- 内容合计：55%
- 语言合计：30%
- 结构：15%

### 4.2 `task1` 小作文

| 维度 | 权重 |
|------|------|
| `task_achievement` | 35% |
| `content_quality` | 20% |
| `structure` | 15% |
| `vocabulary` | 10% |
| `grammar` | 10% |
| `expression` | 10% |

汇总导向：

- 内容合计：55%
- 语言合计：30%
- 结构：15%

说明：

- `task1` 更强调任务完成、对象意识、语气得体和格式规范。
- `task2` 更强调材料解读、观点推进和论述质量。
- 首版为便于系统一致性，两类任务整体保持相近权重结构。

## 五、`vocabulary` 与 `expression` 的边界

### `vocabulary`

重点看：

- 选词准确性
- 词汇丰富度
- 拼写
- 固定搭配和词汇搭配质量

### `expression`

重点看：

- 句式自然度
- 语体正式度
- 逻辑连接是否自然
- 是否有明显中式表达
- 是否存在模板腔、翻译腔、生硬书面语

### 判分规则

- 同一问题不应在两个维度重复作为主要扣分依据。
- 选词、拼写、搭配优先归入 `vocabulary`。
- 生硬、失礼、口语化、模板化、中式英语优先归入 `expression`。

## 六、任务类型差异说明

### 6.1 `task1`

评分重点：

- 写作目的是否明确
- 收信对象和身份关系是否匹配
- 语气是否得体
- 要点是否覆盖
- 格式是否基本正确
- 是否像一篇真实可用的书信 / 邮件 / 通知 / 备忘录

### 6.2 `task2`

评分重点：

- 是否完成“描述材料 + 解读含义 + 给出评论”三个动作
- 是否真正使用了题目材料，而不是脱离材料泛泛而谈
- 是否有较清晰的论证推进
- 是否体现考研层级的书面议论文表达

## 七、通用降档原则

以下规则优先于权重计算：

| 情况 | 处理原则 |
|------|----------|
| 严重跑题 | `task_achievement` 最高 E，总体不得进入高档 |
| 只部分完成任务 | `task_achievement` 不得高于 C |
| `task2` 未描述材料就直接评论 | `task_achievement` 不得高于 C |
| `task1` 缺少核心功能动作 | `task_achievement` 不得高于 C |
| 字数严重不足 | `task_achievement` 与 `content_quality` 同时降档 |
| 语言明显模板化且脱离任务 | `expression` 不得高于 C |

说明：

- 字数具体阈值不写入 Rubric 本体。
- 字数达成率、处罚档位、封顶规则统一由 `exam_policy` 配置决定。
- Rubric 只定义原则，不承载易变参数。

## 八、各维度五档标准

### 8.1 `task_achievement`

#### `task1`

| 等级 | 标准 |
|------|------|
| A | 完整完成题目要求的全部动作，身份、语气、目的、信息都准确，格式规范，读者能直接理解并采取行动 |
| B | 完成了大部分任务要求，只有少量次要信息缺失或表达不够充分，格式和语气基本正确 |
| C | 完成了核心任务，但有明显遗漏，或格式、语气、信息组织存在可见问题，仍能看出写作目的 |
| D | 只完成部分任务，遗漏关键动作，语气或格式不合适，读者较难准确理解作者意图 |
| E | 基本未完成题目任务，内容与题目要求关系很弱，或功能写作属性几乎缺失 |

#### `task2`

| 等级 | 标准 |
|------|------|
| A | 完整完成“描述材料、解释含义、给出评论”三步，材料使用充分，评论与材料紧密对应 |
| B | 三步基本完成，但某一步展开略弱，材料使用较充分，评论基本贴合题意 |
| C | 能完成题目主要要求，但通常存在“描述不充分”或“评论泛化”问题，仍属于可判的合格作文 |
| D | 只完成一部分任务，常见情况是简单描述后直接空泛议论，或只评论不解释材料 |
| E | 严重偏离题目要求，几乎没有完成材料解读任务，或整篇与题意关系很弱 |

### 8.2 `content_quality`

| 等级 | 标准 |
|------|------|
| A | 内容充实，信息有效，展开充分，有较成熟的分析或说明能力；`task2` 能从材料中提炼核心意义，`task1` 能给出有用且具体的信息 |
| B | 内容较完整，有一定展开，论述或说明基本到位，虽不够深入但整体有效 |
| C | 内容基本够用，但展开不足，常停留在表面，细节和支撑不够，属于典型合格线水平 |
| D | 内容贫弱，信息重复或空泛，缺少有效支撑，读者能看懂但收获有限 |
| E | 内容严重不足，空话套话居多，几乎没有实质信息或有效观点 |

### 8.3 `structure`

| 等级 | 标准 |
|------|------|
| A | 篇章组织清晰，开头、主体、结尾功能明确，段落推进自然，衔接得体；`task1` 信息顺序合理，`task2` 论述层次分明 |
| B | 结构较清楚，段落分工基本明确，衔接大致自然，偶有生硬之处但不影响整体阅读 |
| C | 有基本结构，但段落安排和衔接较机械，逻辑推进一般，读者需要一定努力才能跟上 |
| D | 结构松散，信息顺序不理想，段落功能不明确，衔接较差，阅读体验受损 |
| E | 基本无结构可言，句子堆砌，逻辑混乱，难以判断篇章组织意图 |

### 8.4 `vocabulary`

| 等级 | 标准 |
|------|------|
| A | 词汇选择准确、得体、书面化，搭配自然，有一定学术或正式表达能力，重复少，基本无明显低级词汇问题 |
| B | 词汇总体较准确，有一定书面表达意识，偶有搭配或选词问题，但不影响整体质量 |
| C | 词汇量基本够用，但偏常见、偏保守，重复较明显，书面感一般，偶有搭配错误 |
| D | 词汇单一，重复明显，常出现不地道搭配或中式选词，影响表达效果 |
| E | 词汇问题严重，选词频繁失误，搭配大量不当，明显妨碍理解 |

### 8.5 `grammar`

| 等级 | 标准 |
|------|------|
| A | 语法控制稳定，复杂句、从句、非谓语等结构使用自然，错误极少且不影响理解 |
| B | 语法总体较稳，能使用一定复杂结构，有少量错误，但整体清楚流畅 |
| C | 语法基础可接受，简单句为主，复杂结构控制一般，有一些明显错误，但整体仍可理解 |
| D | 语法问题较多，句法控制不稳，复杂结构失误频繁，已明显拖累文章质量 |
| E | 语法错误密集，句子经常失衡或难以成立，阅读理解受到严重影响 |

### 8.6 `expression`

| 等级 | 标准 |
|------|------|
| A | 表达自然、正式、成熟，整体文风符合考研写作场景，几乎无明显中式表达或模板痕迹 |
| B | 表达基本自然，文风较正式，偶有生硬句子，但整体符合考研写作要求 |
| C | 表达可理解，但书面感一般，略显模板化或中式，属于常见合格作文水平 |
| D | 表达较生硬，不自然句子较多，中式表达和套话明显，影响整体印象 |
| E | 表达十分别扭，大量直译、套话或不合场景措辞，整体不符合考研写作要求 |

## 九、任务类型补充锚点

### 9.1 `task1` 补充锚点

| 项目 | 高档表现 | 低档表现 |
|------|----------|----------|
| 格式 | 称呼、结尾、身份关系基本自然 | 缺少基本格式或格式混乱 |
| 目的 | 一开头就能看出写作意图 | 读完仍不清楚为何写这篇 |
| 要点 | 全部覆盖并有必要补充 | 关键要点缺失 |
| 语气 | 与对象和场景匹配 | 过于口语、失礼或不合身份 |

### 9.2 `task2` 补充锚点

| 项目 | 高档表现 | 低档表现 |
|------|----------|----------|
| 描述材料 | 准确抓住图画 / 图表核心特征 | 只笼统提一句，甚至不描述 |
| 解读含义 | 能从材料提炼出中心意义 | 只停留在表面描述 |
| 给出评论 | 评论与材料强相关，有展开 | 空泛套话，与材料脱节 |
| 文章推进 | 描述、解读、评论衔接自然 | 三部分失衡或跳跃明显 |

## 十、Prompt 使用规则

模型在使用本 Rubric 时，必须遵守：

- 先判 `task_achievement`
- 再判 `content_quality` 和 `structure`
- 最后看语言维度
- 任务未完成时，语言好不能强行拉高总评
- `task1` 不按议论文标准打分
- `task2` 不按功能写作标准打分
- 若 `taskType` 缺失，只能做低置信兜底，不应用于高确定性高分判定
- 权重仅用于总分汇总，不可覆盖方向门槛和封顶规则
