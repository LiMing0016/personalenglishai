# 润色模块当前阶段实现说明

## 摘要

当前阶段把全文润色收口为一件事：

**让评分和润色严格使用同一套 rubric、同一套题目上下文、同一套评分逻辑。**

用户链路固定为：

1. 写作文
2. 选择润色档位
3. 系统按当前 rubric 生成 1 个候选稿
4. 系统用同一评分器做 1 次安全复评
5. 用户再提交正式评分，得到与润色目标一致的结果

当前阶段已移除：

- plan / 教练流
- 复杂定制聊天
- 多轮自动教练
- 自动无限重试

右侧 `范文` 面板继续独立存在，作为学习参考，不参与润色兜底。

## 核心原则

### 1. 润色和评分共享同一套事实来源

评分与润色必须共享以下上下文：

- `studyStage`
- `writingMode`
- `taskType`
- `topicContent`
- `taskPrompt`
- `minWords`
- `recommendedMaxWords`
- `rubricKey`

### 2. 润色档位只绑定目标 Band

润色层不再维护第二套分数区间，直接复用 `scoring rubric` 中已有的总分档位区间。

当前默认目标：

- `basic` -> `Band 3`
- `steady` -> `Band 4`
- `advanced` -> `Band 4` 高位 / 接近 `Band 5`
- `perfect` -> `Band 5`

实现上只保存“目标 Band”，不再单独维护“60分 / 90分”这类第二套规则。

### 3. 候选稿必须通过同 rubric 复评

润色不做自动无限重试，但必须做 1 次安全复评。

若候选稿出现以下任一回退，则不允许替换正文：

- 总分下降
- Band 下降
- 核心维度下降
- `exam` 下 `relevance / taskCompletion / coverage` 下降

前端必须明确提示：

`候选稿未通过安全复评，已保留原文。`

## 上下文来源规则

考试模式下固定拆分：

- `topicContent`
- `taskPrompt`

来源规则：

- 图画作文：`topicContent = image_description`
- 材料作文：`topicContent = material_text`
- 任务型/书信类：`topicContent = prompt_text`
- `taskPrompt` 只表示动作要求

这样才能保证：

- 润色时按这道题在改
- 评分时按这道题在判
- 两边不会因为题目来源不同而打架

## 当前润色流程

### 1. 基线评分

系统先用当前 rubric 对原文做 baseline 评分，读取：

- `finalOverall`
- `examBand`
- `dimensionGrades`
- `directionAssessment`
- `capScore / flags / reasons`

### 2. 单次候选稿生成

系统按以下硬约束生成 1 个候选稿：

- 当前学段
- 当前模式
- 当前题目内容
- 当前写作要求
- 当前评分 rubric
- 当前档位目标 Band
- 当前字数区间

### 3. 单次安全复评

候选稿生成后，立即使用同一套评分服务再次评分。

若候选稿不差于原文，则允许展示并替换；若更差，则保留原文。

## 当前前端产品形态

当前阶段前端只强调两条能力：

- `自动润色`
- `范文`

`自动润色` 面板只展示这些核心信息：

- 当前档位
- 当前策略
- baseline / final band
- 当前候选稿是否通过安全复评
- 若失败，明确说明已回退原文

当前实现只保留自动润色主流程，不再暴露任何 plan / 教练式状态流。

## 范文面板

右侧原 `结构` 面板已替换为 `范文` 面板，内部仍保留 `panel = 'structure'` 作为状态键，以减少主链路改动。

当前 `范文` 面板固定生成两张学习卡：

- `优秀作文`
- `满分作文`

每张卡都包含：

- 范文正文
- `高分原因`
- `你该怎么做`

该能力独立于润色主链路，不承担润色结果兜底。

## 配置与扩展

当前实现中：

- `scoring rubric` 负责评分标准
- `polish rubric` 只负责档位目标、目标 Band、路由与安全守门

后续新增学段时，目标是只替换：

- `scoring rubric`
- `polish rubric`
- prompt 模板
- model adapter

不改主流程代码。

## 当前验证重点

推荐重点验证：

1. 同一篇 `exam` 作文：
   - 润色前评分一次
   - 润色后评分一次
   - 两次评分都使用同一 rubric 和同一题目上下文
2. 同一篇 `free` 作文：
   - 润色和评分使用同一 free rubric
3. 图画/材料/任务型作文：
   - 评分与润色读取同一 `topicContent`
4. 候选稿更花哨但分数更低：
   - 不替换正文
5. 四个润色档位：
   - 都能对应到明确的目标 Band

## 默认假设

- 当前阶段的重点是“评分与润色对齐”，不是“聊天体验升级”。
- `scoring rubric` 已经包含 Band 区间，润色层不再维护第二套分数体系。
- 润色档位只维护 `targetBand`。
- 在这套一致性稳定之前，不继续扩大 plan、教练、多轮提分或自动闭环的范围。
