# 润色功能需求文档

## 1. 功能概述

用户完成作文后，系统提供 **3 个档次的润色版本**，每个档次对标不同的考试分数段。系统先评估当前作文水平，再基于目标分数进行润色，确保只往上提升、不会降级。

润色结果以**全文对比**的形式呈现，用户可以逐句查看改动，学习每处修改的原因。

---

## 2. 核心概念

### 2.1 三个润色档次

| 档次 | 名称 | 定位 |
|------|------|------|
| 1 | 基础版 | 当前水平 + 一个台阶，修正错误，表达自然，确保扣题 |
| 2 | 进阶版 | 当前水平 + 两个台阶，升级词汇句式，优化结构衔接 |
| 3 | 满分版 | 对标该考试满分标准，范文级别 |

### 2.2 分数锚定机制

润色前先评估当前作文的近似分数，3 个档次动态生成：

**示例（高考 150 分制，当前作文约 105 分）：**

| 档次 | 目标分数 |
|------|---------|
| 基础版 | ~115 分 |
| 进阶版 | ~130 分 |
| 满分版 | ~145 分 |

**示例（雅思 9 分制，当前作文约 5.5 分）：**

| 档次 | 目标分数 |
|------|---------|
| 基础版 | ~6.0 分 |
| 进阶版 | ~7.0 分 |
| 满分版 | ~8.0+ 分 |

规则：
- 所有档次 **≥ 当前分数**，不会向下润色
- **满分版**有固定天花板（考试最高分或实际可达最高分）
- 当前水平越高，档次间差距越小
- 若当前作文已接近满分，基础版和进阶版可合并或提示"已接近满分"

### 2.3 参考评分维度（e-rater 框架）

润色时 AI 按以下维度分档次侧重：

| 维度 | 说明 | 基础版 | 进阶版 | 满分版 |
|------|------|--------|--------|--------|
| Grammar 语法 | 语法正确性 | ● | ● | ● |
| Usage 用法 | 词语搭配、介词使用 | ● | ● | ● |
| Mechanics 机制 | 拼写、标点、大小写 | ● | ● | ● |
| Topical Analysis 扣题度 | 是否回应题目要求 | ● | ● | ● |
| Style 文体 | 句式多样性 | | ● | ● |
| Organization 组织 | 篇章结构完整性 | | ● | ● |
| Vocabulary 词汇 | 词汇复杂度和丰富度 | | ● | ● |
| Development 展开 | 论点展开深度 | | | ● |

- **基础版**：重点修 Grammar/Usage/Mechanics + 扣题
- **进阶版**：以上 + Style/Organization/Vocabulary
- **满分版**：全部维度拉满，特别是 Development

### 2.4 润色约束

无论哪个档次，都遵守以下约束：
- **保留用户原有观点和论证方向**，不替换核心立场
- **不编造事实或数据**，补充的例子需合理
- **低档次不动结构**（基础版保留原文段落划分）
- **高档次可调整结构**（进阶版/满分版可重组段落、补充论据）

## 3. 用户流程

### 3.1 入口

1. **语法检查完成后引导**：语法检查无错误 → 显示"语法无误，试试润色提升表达？" → 点击进入润色面板
2. **ToolRail 直接进入**：点击右侧工具栏"润色"按钮
3. **评分结果引导**：评分完成后，建议用户润色提升

### 3.2 主流程

```
用户点击"润色"
  → 选择考试类型（高考/四级/六级/雅思/托福/...）（若已设置则跳过）
  → 系统显示"正在分析作文水平..."
  → AI 评估当前作文近似分数
  → 生成 3 个档次卡片，显示目标分数范围
  → 用户选择一个档次
  → 系统显示"正在润色..."（带进度状态）
  → 返回润色结果
  → 用户查看全文对比（原文 vs 润色版）
  → 可逐句查看修改原因
  → 可选择"采用此版本"替换原文 或 继续查看其他档次
```

### 3.3 考试类型

系统需支持的考试类型及对应分制：

| 考试 | 满分 | 及格线 | 说明 |
|------|------|--------|------|
| 高考英语 | 150 | 90 | 全国卷 |
| 中考英语 | 因地区而异 | - | 按地区配置 |
| CET-4 | 710（写作106.5） | 63.9 | 大学英语四级 |
| CET-6 | 710（写作106.5） | 63.9 | 大学英语六级 |
| 考研英语 | 20（小作文10+大作文10） | - | 硕士研究生入学 |
| 雅思 IELTS | 9.0 | - | 写作单科 |
| 托福 TOEFL | 30 | - | 写作单科 |
| 自由写作 | 无 | - | 不对标考试，按 CEFR 等级润色 |

自由写作模式下，档次改为 CEFR 等级：基础版 → B1、进阶版 → B2、满分版 → C1。

## 5. 后端设计

### 5.1 API 接口

#### 5.1.1 评估作文水平

```
POST /api/writing/polish/assess
```

请求：
```json
{
  "text": "作文原文...",
  "examType": "gaokao",
  "taskPrompt": "题目要求..."
}
```

响应：
```json
{
  "estimatedScore": 105,
  "maxScore": 150,
  "tiers": [
    { "tier": "basic",    "targetScore": 115, "label": "基础版", "description": "修正错误，表达自然" },
    { "tier": "advanced",  "targetScore": 130, "label": "进阶版", "description": "升级词汇，优化结构" },
    { "tier": "perfect",  "targetScore": 145, "label": "满分版", "description": "范文级别，全面提升" }
  ]
}
```

#### 5.1.2 执行润色

```
POST /api/writing/polish/execute
```

请求：
```json
{
  "text": "作文原文...",
  "examType": "gaokao",
  "taskPrompt": "题目要求...",
  "tier": "advanced",
  "targetScore": 130,
  "estimatedScore": 105
}
```

响应：
```json
{
  "polished": "润色后全文...",
  "changes": [
    {
      "type": "grammar",
      "original": "She have a apple.",
      "revised": "She has an apple.",
      "reason": "主谓一致：She 搭配 has；元音前用 an",
      "span": { "start": 0, "end": 18 }
    },
    {
      "type": "vocabulary",
      "original": "learn knowledge",
      "revised": "acquire knowledge",
      "reason": "搭配更地道：acquire knowledge 比 learn knowledge 更自然",
      "span": { "start": 45, "end": 60 }
    }
  ],
  "summary": {
    "grammarFixes": 3,
    "vocabularyUpgrades": 5,
    "styleFixes": 2,
    "contentAdditions": 1
  }
}
```

### 5.2 AI Prompt 结构

```
系统 prompt:
- 你是英语写作润色助手
- 考试类型：{examType}，满分 {maxScore} 分
- 题目要求：{taskPrompt}
- 当前作文水平：约 {estimatedScore} 分
- 目标水平：{targetScore} 分
- 润色档次：{tier}
- 参考评分维度：{根据档次选择的 e-rater 维度}
- 约束：保留用户核心观点，不编造事实，{档次特定约束}

用户 prompt:
- 请润色以下作文，并逐条列出改动及原因：
- {原文}
```

## 6. 技术要点

### 6.1 性能

- **评估接口**：需快速返回（< 3s），可用轻量 prompt 或规则评估
- **润色接口**：耗时较长（5-15s），前端需有进度状态
- 两个接口分开，评估结果可缓存（同一篇文章不重复评估）

### 6.2 成本控制

- 每次润色调一次 GPT-4o，输入 = 系统 prompt + 全文，输出 = 润色版 + changes JSON
- 用户可能连续试 3 个档次 → 最多 3 次 API 调用 + 1 次评估
- 可考虑评估用 GPT-4o-mini 降成本

### 6.3 与现有功能的关系

| 现有功能 | 关系 |
|---------|------|
| 语法检查（LT + Sapling） | 润色前建议先完成语法检查，减少低级错误 |
| 写作评分（evaluate） | 评分是打分 + 反馈，润色是直接给改写版本，互补 |
| AI 改进建议（suggestions） | 建议是点状修改，润色是全文级改写，润色包含建议 |
| 现有 RewritePanel | 需要重构，当前只支持逐条 suggestion 润色 |

## 7. 分期实施建议

### Phase 1：基础可用
- 后端：评估接口 + 润色接口（单个 AI 调用）
- 前端：档次选择 → 润色中 → 结果展示（纯文本对比）
- 支持高考 + 雅思两种考试类型

### Phase 2：体验优化
- 逐句 diff 对比 + 高亮
- 编辑器联动（点击改动跳转）
- 改动分类摘要
- 支持更多考试类型

### Phase 3：高级功能
- 用户画像推荐默认档次
- 润色历史记录
- "采用部分改动"（逐句选择是否采纳）
- 润色前后分数对比可视化
