# 考试模式 — 题目设置页设计方案

## 概述

用户选择"考试模式"后，进入独立的**题目设置页**，完成题目信息填写后再进入写作编辑器。支持三种题目来源：手动输入、AI 生成、历年真题。

---

## 核心规则

1. **题目必填**：考试模式下必须有题目，否则提示切换到自由写作模式
2. **体裁/字数可选**：不填则点"开始写作"时由 GPT 自动推断
3. **前端优先校验**：纯数字、过短文本等前端直接拦截，减少 GPT 调用
4. **GPT 兜底**：用户点"开始写作"时，缺失信息由 GPT 一次性补全推断
5. **信任用户**：不做过度防御，正常学习的用户不会乱填

---

## 页面结构

### 入口流程

```
WritingPage 模式选择
    |
    +-- 自由写作 --> 直接进入 EditorShell
    |
    +-- 考试写作 --> 进入题目设置页（ExamSetup）
                        |
                        +-- 填写/选择题目
                        +-- 点击"开始写作" --> 进入 EditorShell（携带结构化题目）
                        +-- 点击"返回" --> 回到模式选择
```

### 三个 Tab

#### Tab 1: 手动输入（第一期实现）

```
┌──────────────────────────────────────────────┐
│  题目/情境描述 *                              │

│  └────────────────────────────────────┘      │
```

**验证逻辑：**
- 题目为空 → 禁用"开始写作"按钮
- 纯数字/特殊字符 → 前端提示"请输入有效的作文题目"
- 长度 < 5 字 → 提示"题目过于简略，建议补充写作情境"
- 体裁/字数未选 → 允许提交，GPT 在评分时自动推断

## 必要信息项

| 信息项 | 是否必填 | 缺失处理 |
|--------|----------|----------|
| 题目/情境描述 | 必填 | 不允许为空，提示输入或切换自由模式 |
| 体裁 | 可选 | 不选则 GPT 在评分时自行推断 |
| 字数范围 | 可选 | 不选则 GPT 在评分时自行推断 |

---

## 数据流

```
ExamSetup 页面
    |
    v
用户填写/选择题目信息
    |
    v
点击"开始写作"
    |
    v
组装 taskPrompt 字符串（题目 + 体裁 + 字数）
    |
    v
传入 EditorShell.initialTaskPrompt
    |
    v
评分时作为 taskPrompt 字段发送给后端
```

### taskPrompt 组装规则

```
输入: { topic: "给Tom写信介绍春节", genre: "书信", wordRange: "80-120" }
输出: "给Tom写信介绍春节\n体裁：书信\n字数要求：80-120词"

输入: { topic: "给Tom写信介绍春节", genre: null, wordRange: null }
输出: "给Tom写信介绍春节"
```

## 技术实现（第一期：手动输入）

### 前端

**修改文件**: `web/src/pages/app/WritingPage.vue`
- `Phase` 类型新增 `'exam-setup'`
- 选择考试模式 → `phase = 'exam-setup'`
- ExamSetup 组件 emit `confirm` 事件，携带结构化题目
- 确认后 → `phase = 'editor'`，传入 `initialTaskPrompt`

**新建文件**: `web/src/pages/app/ExamSetupPage.vue`
- 题目 textarea + 体裁按钮组 + 字数按钮组
- 前端基础校验（非空、非纯数字、最小长度）
- emit: `confirm({ topic, genre, wordRange })`, `back()`

### 后端

第一期不需要新 API。题目信息组装成 `taskPrompt` 字符串，复用现有评分接口。
