# 评分反馈质量方法论

> 本文档定义单次评分的反馈质量标准。
> 目标：让每一次评分反馈都像一位有经验的老师亲笔写的批注——专业、具体、温暖、有行动力。
> 所有改进均在 Prompt 层实现，不需要前端或数据库改动。

---

## 一、核心原则

### 1.1 一次评分就要让学生觉得"这个老师真的读了我的作文"

ChatGPT 式反馈的致命问题不在于不准确，而在于**泛泛而谈**：
- "Your grammar needs improvement" — 哪里？什么错误？
- "Good vocabulary usage" — 哪个词用得好？好在哪里？
- "Try to vary your sentence structures" — 怎么变？变成什么样？

**我们的标准：每一句反馈都必须能在原文中找到对应的证据。**

### 1.2 三个不可妥协的质量红线

| 红线 | 要求 |
|------|------|
| **具体性** | 反馈中不得出现无法对应原文的空泛描述 |
| **温度感** | 反馈语言不得像评分表/测评报告，必须像人在说话 |
| **可行性** | 每个改进建议必须是学生"现在就能做"的，不是"以后努力" |

## 二、评分锚点校准

### 2.1 等级含义的精确定义

AI 打分不稳定的根本原因是 **Prompt 中等级描述不够具体**。
模糊描述导致 AI 在 B 和 C 之间来回摇摆。

**校准原则：**

```
C 是基准线 = "一个普通高中生能写出来的水平"
B = 在 C 的基础上有明显的亮点
A = 亮点突出，让老师忍不住想划线表扬
D = 问题明显，但还能读懂
E = 基本不可读
```

### 2.2 错误密度 → 等级关联

仅靠主观描述不够，需要给 AI 可量化的锚点：

| 维度 | A | B | C | D | E |
|------|---|---|---|---|---|
| **grammar** | 0-1 错误/100词 | 2-3 错误/100词 | 4-5 错误/100词 | 6-8 错误/100词 | >8 错误/100词 |
| **vocabulary** | 高级词汇占比 >20% | 10-20% | 5-10% | <5%，大量重复 | 基本词汇也用错 |
| **structure** | 段落清晰 + 过渡自然 | 段落清晰 + 过渡基本 | 有分段，过渡生硬 | 不分段或逻辑混乱 | 无结构可言 |

> 注意：这些是指导性锚点，不是硬规则。AI 应综合判断，但在边界情况下以这些锚点为准。

### 2.3 Prompt 中如何注入锚点

```
在评分时，请参考以下量化锚点辅助判断：

grammar 维度：
  - A: 每 100 词不超过 1 处语法错误，句式丰富（含从句、虚拟语气等）
  - B: 每 100 词 2-3 处轻微错误，句式有变化
  - C（基准线）: 每 100 词 4-5 处错误，以简单句为主，偶有复合句
  - D: 每 100 词 6-8 处错误，中式英语特征明显
  - E: 语法错误遍布，难以理解

如果某个维度在两个等级之间犹豫，请选择**较低的等级**。
宁可偏严，因为学生更需要知道差距，而不是虚假安慰。
```

## 三、优点反馈规范

### 3.1 核心要求：引用原文

**错误示范：**
```
优点：词汇运用不错，表达较为流畅。
```
→ 学生看完不知道自己哪里"不错"。

**正确示范：**
```
优点：你用了 "safeguard biodiversity" 而不是简单的 "protect nature"，
这个词汇选择非常准确，体现了学术表达的意识。
第二段用 "not only...but also" 引导的并列结构让论点更有说服力。
```
→ 学生知道自己哪里做得好，以后会刻意重复。

### 3.2 Prompt 指令

```
【优点反馈要求】
每个优点必须满足：
1. 引用原文中的具体词汇或句子（用引号标出）
2. 说明这个词汇/句子好在哪里（不是"用得好"，而是"好在哪里"）
3. 如果学生用了超出其当前水平的表达，明确指出"这超出了 X 级的通常水平"

示例格式：
  strength: "你用了 'shed light on' 来替代 'explain'，这是一个高级短语动词，表达更地道。"
  quote: "This phenomenon sheds light on the importance of..."
```

### 3.3 等级与优点的关系

| 等级 | 优点反馈策略 |
|------|-------------|
| A/B | 指出精彩之处，鼓励保持，可以提到"这个表达是 X 级水平" |
| C | 找到相对好的地方，给出"如果这样改会更好"的提升方向 |
| D/E | 找到任何一个正确的表达并肯定，避免全盘否定 |

> D/E 级学生最需要鼓励。哪怕只有一句话用对了时态，也要指出来。

## 四、缺点反馈规范

### 4.1 核心要求：一步到位法

**不要说"你的语法有问题"，要说"你的语法错在哪里，正确的是什么"。**

**错误示范：**
```
缺点：语法错误较多，需要加强基础语法学习。
```
→ 学生知道自己语法不好，但不知道怎么改。

**正确示范：**
```
缺点：你写了 "I very like this activity"，
这是一个典型的中式英语表达。
英语中 "very" 不能直接修饰动词 "like"，正确说法是 "I really enjoy this activity"
或 "I like this activity very much"。
```
→ 学生立刻知道错在哪里、为什么错、正确的是什么。

### 4.2 Prompt 指令

```
【缺点反馈要求】
每个缺点必须包含三个部分（缺一不可）：

1. 原文引用：学生写了什么（用引号标出原文）
2. 错误解释：为什么这样写不对，简要说明原因
3. 正确示范：应该怎么写，给出可以直接替换的正确表达

格式：
  weakness: "你写了「原文」，[解释为什么不对]。正确的写法是「正确表达」。"

注意：
- 如果这个错误是中式英语特征，请明确指出"这是中式英语"
- 如果这个错误在之前的评分中出现过（根据 ability profile），请加上"这个问题在你之前的作文中也出现过"
- 不要只列出错误，必须给出修正后的完整表达
```

### 4.3 缺点数量控制

| 等级 | 最多列出缺点数 | 原因 |
|------|---------------|------|
| A/B | 2-3 | 已经很好，指出精细化改进点 |
| C | 3-4 | 聚焦最关键的几个问题 |
| D/E | 2-3（仅最严重的）| 太多会打击信心，只抓最要紧的 |

> 原则：**列出的每个缺点，学生都必须能在下一篇作文中改正。**
> 如果一个缺点需要"长期积累"才能改正（如"词汇量不足"），不要列为缺点，
> 而是在总结中作为建议提出。

## 五、总结（Summary）规范

### 5.1 核心要求：像老师的手写评语

总结不是各维度评分的汇总，而是一段**有温度的、针对这篇作文的整体评价**。

**错误示范：**
```
总分 10/15，语法 C，词汇 B，结构 C。整体中等水平，需要在语法方面加强练习。
```
→ 这是评分表，不是评语。

**正确示范：**
```
这篇作文的话题选择很好，你对环保问题的观点清晰有力。
特别是 "safeguard biodiversity" 这个表达让我眼前一亮。

但语法上的小毛病拖了后腿——主要是时态不一致的问题，
第一段用过去式，第二段突然切到现在式，读起来有点跳跃。
这不是大问题，下次写完检查一遍时态就能避免。

下一步建议：写完后大声读一遍你的作文，
凡是你觉得"读起来别扭"的地方，往往就是时态或搭配出了问题。
```

### 5.2 总结的三段结构

```
第一段：肯定（找到这篇作文最大的亮点，具体说出来）
第二段：主要问题（最多 1-2 个核心问题，不是罗列所有问题）
第三段：下一步（一个具体的、可以"今天就做"的行动建议）
```

### 5.3 Prompt 指令

```
【总结要求】
写一段 100-150 字的中文总结，语气像一位友善的英语老师在作文本上写的评语。

结构：
1. 第一句话肯定这篇作文最大的优点（引用原文中的具体词句）
2. 指出 1-2 个最关键的改进点（不要罗列所有问题）
3. 最后给出一个具体的、可操作的下一步建议
   （不是"多读英语文章"这种泛泛建议，
     而是"下次写完后检查所有的时态是否一致"这种可执行的）

禁止出现以下词汇：
  - "总体来说"（像评分报告，不像人在说话）
  - "建议加强"（太官方）
  - "有待提高"（敷衍）
  - "整体水平中等"（冰冷）

语气：温暖但专业，像一个学生喜欢的英语老师。
```

## 六、AI 输出 Schema 增强

### 6.1 当前 Schema 的不足

当前 `analysis` 字段结构：

```json
{
  "analysis": {
    "grammar": {
      "strength": "...",
      "weakness": "...",
      "suggestion": "..."
    }
  }
}
```

问题：
- `strength` 和 `weakness` 是纯文本，没有强制引用原文
- 没有 `quote` 字段来要求 AI 提供原文证据
- `summary` 与 `analysis` 脱节

### 6.2 增强后的 Schema

```json
{
  "analysis": {
    "grammar": {
      "strength": "你用了 'had I known' 倒装结构，这是 A 级水平的表达",
      "strength_quote": "Had I known the consequences, I would have acted differently.",
      "weakness": "第二段时态从过去式突变为现在式",
      "weakness_quote": "Yesterday I went to the park. I see many flowers there.",
      "suggestion": "写完后检查每个段落的时态是否统一"
    }
  },
  "summary": "这篇作文...（三段式评语）",
  "priorityFocus": {
    "dimension": "grammar",
    "reason": "你的语法错误集中在时态不一致，这是高考扣分最多的类型",
    "actionItem": "下次写完后，用不同颜色的笔标出每句话的时态，检查是否一致"
  }
}
```

### 6.3 新增 `priorityFocus` 字段

每次评分只给出 **一个** 最优先改进的维度，包含：
- `dimension`：哪个维度
- `reason`：为什么是这个维度（与高考/考试的关联）
- `actionItem`：一个具体的行动项

> 设计理由：学生一次只能改一个问题。给 5 个建议等于没给建议。

## 七、反馈语言风格指南

### 7.1 用词对照表

| 避免 | 使用 |
|------|------|
| 总体来说 | 这篇作文... / 读完你的作文... |
| 有待提高 | 如果这样改会更好 |
| 建议加强 | 下次试试... / 你可以... |
| 整体中等 | 你已经掌握了基础，下一步是... |
| 词汇量不足 | 你现在常用的词是 X/Y/Z，试试用 A/B/C 替换 |
| 语法错误较多 | 我注意到有 N 处时态/主谓一致的问题 |

### 7.2 鼓励策略

对不同水平的学生使用不同的鼓励方式：

```
D/E 级：
  "你用对了 'because' 来连接原因，说明你已经有了因果逻辑的意识，
   这是很好的起点。"
  → 找到微小的正确点并放大

C 级：
  "你的作文结构清晰，有开头有结尾有论点，
   这说明你的写作框架已经建立起来了。
   下一步就是让每个论点更有说服力。"
  → 肯定框架，引导细化

A/B 级：
  "你这篇作文已经超出了普通高中生的水平，
   'shed light on' 这个表达很地道。
   如果想继续提升，可以尝试更多长难句的使用。"
  → 肯定超越，指向更高目标
```

## 八、质量验证清单

每次评分反馈上线前，用以下清单检验：

- [ ] **优点**是否引用了原文具体词句？
- [ ] **缺点**是否包含"原文 → 解释 → 正确写法"三部分？
- [ ] **总结**是否像老师在说话，而不是评分报告？
- [ ] 总结是否包含一个**今天就能做**的行动建议？
- [ ] D/E 级作文是否找到了至少一个肯定点？
- [ ] 是否列出了超过 4 个缺点？（如果是，删到 3-4 个）
- [ ] `priorityFocus` 是否给出了**一个**最优先改进方向？
- [ ] 反馈中是否出现了"总体来说""有待提高"等禁用词汇？

## 十、实施路径

### 第一步：Prompt 改写（无代码改动）

将本文档中的 Prompt 指令片段整合到 `WritingEvaluateMockService` 的 SYSTEM_PROMPT 中：
- 优点反馈要求（引用原文）
- 缺点反馈要求（三段式）
- 总结要求（三段结构 + 禁用词汇）
- 量化锚点（辅助判分）

### 第二步：Schema 增强（小量代码改动）

- `WritingEvaluateResponse` 增加 `strength_quote`、`weakness_quote` 字段
- `priorityFocus` 对象新增 `actionItem` 字段
- 前端 ScorePanel 展示增强内容

### 第三步：效果验证

- 收集 20 篇不同水平的作文样本
- 对每篇分别用旧 Prompt 和新 Prompt 评分
- 邀请 2-3 位英语老师盲评两组反馈的质量
- 根据反馈微调 Prompt 措辞

> 评分反馈的质量不是一次能做到位的，需要持续迭代。
> 但方向是确定的：**每一句话都要有证据，每一个建议都要能执行。**
