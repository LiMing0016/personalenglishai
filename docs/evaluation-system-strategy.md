# 评分系统战略文档

> 本文档定义评分系统的战略地位、当前问题、个性化路径和商业化逻辑。
> 是整个产品的核心规划文件。

---

## 一、评分系统是整个产品的根基

### 为什么是根基，不是功能

大多数英语学习产品把"评分"当作一个功能模块。
这个产品不同——**评分是数据采集入口，是所有个性化的起点**。

```
用户提交作文
    ↓
评分系统分析（多维度）
    ↓
写入用户能力档案（user_ability_profile）
    ↓
下一次评分时，Prompt 已知道这个学生的弱点
    ↓
反馈越来越精准 → 用户越来越依赖 → 留存
```

每一次评分，系统都在"认识"这个学生。
用户用得越多，系统对他的了解越深，
这种积累是 ChatGPT 结构性无法替代的。

### 根基体现在三个层面

| 层面 | 说明 |
|------|------|
| **数据层** | 每次评分 = 一条能力记录，积累后形成成长轨迹 |
| **产品层** | 评分质量决定用户是否信任产品，信任是付费前提 |
| **技术层** | 评分 Prompt + Rubric 是核心资产，越积累越难复制 |

## 二、评分系统如何产生收入

### 直接变现

```
免费额度（每月 N 次）→ 触达价值，建立信任
    ↓
订阅转化（月卡/季卡）→ 核心收入
    ↓
高级功能（深度报告/错误库/进步分析）→ 溢价收入
```

### 间接变现（更重要）

评分系统产生的数据，驱动后续所有增值功能：

```
评分 → 发现薄弱点
    ↓
推荐针对性练习（未来功能）→ 更多使用时长
    ↓
追踪进步 → 用户看到成长 → 续费意愿强
    ↓
分享进步报告 → 社交传播 → 新用户获取
```

### 为什么用户愿意为"评分"付费

ChatGPT 可以批改作文，但它：
- 每次标准不一致（同一篇文章打两次分不同）
- 不记得你上次的错误
- 不知道你在备考什么
- 给不了高考维度的精准评分

**你的系统提供的是"一致 + 专业 + 记忆"，这是付费点。**

## 三、当前评分系统的核心问题

### 问题 1：Rubric 数据是占位符（P0）

现状：
```
Grammar A: "Excellent. Fully meets task and uses clear evidence..."
```
这是通用模板，AI 无法据此做出专业判断。

目标：
```
Grammar A（高中水平）：
  句式丰富多样，复杂从句运用自如，几乎无语法错误；
  体现高考写作满分卷水平，能用虚拟语气、倒装句等高级结构；
  偶有笔误但不影响理解。

Grammar B：
  基本语法正确，有少量错误但不影响交流；
  句式有一定变化，简单句和复合句搭配使用；
  时态基本一致，主谓基本一致。

Grammar C（及格线）：
  错误较多，以简单句为主；
  时态混乱或主谓不一致的情况明显；
  中式英语特征突出。

Grammar D：
  错误频繁，严重影响理解；
  大量中式直译，句子结构混乱。

Grammar E：
  几乎无法理解，语法错误遍布全文，
  无完整正确的句子。
```

**这是评分质量的地基，必须先做。**

### 问题 2：错误标注无法高亮（P0）

现状：`ErrorDto` 里的 `span` 全是 `(0, 0)`，前端无法高亮显示错误位置。

原因：AI 没有被要求返回"原文片段"，只返回了建议文字。

解决方案：在 AI 输出 schema 里加 `original` 字段：
```json
"errors": [
  {
    "original": "I very like this activity",
    "suggestion": "I really enjoy this activity",
    "type": "grammar",
    "severity": "major",
    "reason": "中式直译，副词位置错误"
  }
]
```
后端拿到 `original`，在 essay 里做 `indexOf`，计算真实 `start/end`。

### 问题 3：分数体系与高考脱节（P1）

现状：A=90, B=80, C=70... 是百分制，学生看不懂。

高考英语写作实际评分：
| 题型 | 满分 | 档次 |
|------|------|------|
| 应用文（第一节）| 15分 | 13-15 / 10-12 / 6-9 / 1-5 / 0 |
| 读后续写（第二节）| 25分 | 21-25 / 16-20 / 11-15 / 6-10 / 1-5 |

学生需要看到的是："你这篇作文预计得 **11/15 分**，主要扣分在语法和词汇。"

### 问题 4：没有针对中国学生的错误库（P1）

中国高中生有高度规律的错误模式，应写入 prompt 优先检测：

```
【中国高中生高频错误，请优先识别】
1. 中式直译：
   - "very + 形容词" 替代副词短语（very beautiful → extremely beautiful）
   - "have a good time to do" 错误搭配
   - "with the development of society" 万能开头滥用

2. 时态错误：
   - 描述文章内容时用过去时（The author described → describes）
   - 叙事时态不一致

3. 主谓一致：
   - Everyone have / The number of students are

4. 连接词滥用：
   - Firstly... Secondly... Thirdly... Fourthly...（机械堆砌）

5. 应用文格式：
   - 缺少称呼（Dear...）
   - 缺少落款（Yours sincerely,）
   - 段落结构不符合英文书信规范

6. 词汇局限：
   - get/have/make/do 动词过度依赖
   - big/small/good/bad 形容词过度依赖
```

### 问题 5：能力档案没有被激活（P2）

`user_ability_profile` 表已存在，`AbilityPromptBuilder` 也已实现，
但评分完成后**没有回写用户能力数据**。

每次评分后，应该更新：
```
grammar 维度 → 本次得 C，历史均分 C+，趋势：持平
vocabulary 维度 → 本次得 B，历史均分 C，趋势：进步 ↑
```

## 四、个性化的核心逻辑

### 什么是真正的个性化

不是"根据年级调整题目难度"，而是：

> **系统知道你是谁，知道你的弱点，知道你在进步，
> 每一次反馈都是专门为你写的。**

### 个性化的四个维度

#### 维度 1：学段适配（Stage）

不同学段，评分标准完全不同：

| 学段 | 核心目标 | 关注维度 |
|------|----------|----------|
| 初中 | 基础语法 + 简单表达 | grammar, structure |
| 高中（高考）| 应用文/读后续写规范 | task_achievement, expression |
| 大学四六级 | 论说文逻辑 + 词汇丰富度 | content_quality, vocabulary |
| 考研英语 | 图表作文 + 大作文逻辑 | structure, content_quality |
| 雅思/托福 | 学术写作规范 | 全维度，更严格 |

系统已有 `stage` 字段，现在只有 `highschool`，需要扩展。

#### 维度 2：当前水平适配（Level）

同一学段内，初学者和高水平学生需要不同的反馈：

```
低水平学生（D/E 级）：
  → 只指出最严重的 1-2 个问题
  → 反馈语言简单易懂
  → 鼓励为主，避免打击信心
  → "你的开头写得很清晰，下一步专注改语法"

高水平学生（A/B 级）：
  → 指出细节和精细化问题
  → 反馈语言专业
  → 挑战为主，推动更高标准
  → "你的词汇选择得当，但句式变化不够，
     建议尝试更多复合句和倒装结构"
```

`AbilityPromptBuilder` 已经有了这个框架，需要真正充实内容。

#### 维度 3：历史错误追踪（History）

用户第 5 次提交作文时，系统应该知道：
```
你过去 4 次评分的主要问题：
- 主谓一致错误（出现 3 次，本次重点检查）
- 连接词单调（出现 2 次，有改进）
- 词汇局限（首次出现）
```

Prompt 里注入历史弱点 → AI 优先关注这些问题 → 反馈更有连续性。

#### 维度 4：目标导向（Goal）

学生的学习目标不同，关注点也不同：

```
目标：高考冲刺（3个月内）
  → 优先关注高考失分点：格式 + 要点覆盖
  → 每次评分给出"高考预估得分"

目标：长期提高（无时限）
  → 均衡发展各维度
  → 每月生成进步报告

目标：纠正特定问题（如语法）
  → 该维度权重加大
  → 其他维度简评即可
```

## 五、不同学段的 Rubric 规划

### 已实现
- `highschool` → free / exam 两种模式

### 待实现（按优先级）

```
highschool（完善）
  ├── free：自由写作，全面评价
  └── exam：对应高考两种题型
      ├── 应用文（application）
      └── 读后续写（continuation）

middle_school（初中，下一阶段）
  ├── free
  └── exam → 中考写作

college（大学，后续）
  ├── cet4（四级作文）
  ├── cet6（六级作文）
  └── free

postgrad（考研）
  ├── translation（翻译）
  └── writing（大作文/图表）

ielts / toefl（国际考试，高级版本）
```

每个学段都需要独立的 Rubric 数据和专属 Prompt 片段。

## 六、个性化成长路径设计

### 用户的成长旅程

```
第 1 次评分
  → 建立 baseline 能力档案
  → 系统：「你目前语法 C 级，词汇 D 级，结构 C 级」

第 2-5 次评分
  → 每次对比上次，显示变化
  → 系统：「你的语法比上次提升了，词汇还需要努力」

第 10 次评分
  → 生成阶段性报告
  → 「你加入以来提交了 10 篇作文，语法从 D 进步到 C，
     你最稳定的优势是结构，最需要继续加强的是词汇」

第 30 次评分
  → 系统已经非常了解你
  → 「你有一个持续的问题：连接词单调。
     我注意到你第 3、7、12、18 次都用了 Firstly/Secondly，
     本次依然如此。建议专项练习过渡词多样性。」
```

### 技术实现路径

```
评分完成
    ↓
解析各维度 grade（A-E）
    ↓
读取 user_ability_profile（当前档案）
    ↓
计算新的加权均值（新结果权重 30%，历史权重 70%）
    ↓
写回 user_ability_profile
    ↓
下次评分时，AbilityPromptBuilder 读取档案
    ↓
Prompt 携带「该用户历史弱点」信息
    ↓
AI 反馈更有针对性
```

## 八、一句话定位

> **不是批改工具，而是英语写作成长伴侣。**
> 每次评分，系统都更懂你。用得越久，价值越大。
> 这是 ChatGPT 无法替代你的根本原因。
